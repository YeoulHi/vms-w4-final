| 구분                | 기술                          | 역할 및 사용 범위                                                             | 유지보수 기준 충족           |
| ----------------- | --------------------------- | ---------------------------------------------------------------------- | -------------------- |
| **Backend**       | Django 5.x                  | 메인 서버 프레임워크. Admin 업로드·세션 인증·템플릿 렌더링 담당.                               | 안정적 / 하위호환 우수        |
| **API Layer**     | Django REST Framework       | `/api/dashboard/chart-data/` 단일 엔드포인트만 사용. Chart.js 데이터 포맷(JSON)으로 응답. | 인기도 높고 AI 친화적        |
| **DB**            | Supabase (PostgreSQL only)  | DB 호스팅 전용. Django ORM으로 접근. Auth/Storage 기능은 사용 안 함.                   | 활발히 유지보수 / 하위호환 보장   |
| **Excel Parser**  | pandas                      | 엑셀 고정 스키마 파싱 및 DB 저장. 파싱 오류는 콘솔 로그로 출력.                                | AI가 잘 구현 / 성숙한 라이브러리 |
| **Visualization** | Chart.js                    | 메인 대시보드에서 막대·라인·파이 차트 표현.                                              | 대중적 / 버전 안정성 높음      |
| **Frontend(UI)**  | Django Template + Bootstrap | 내부용 간단한 대시보드 화면 구성. SPA 미사용.                                           | 유지보수 용이 / 최소 복잡도     |
| **Deployment**    | Railway                     | 연습용 배포 플랫폼. 환경 종속 없음. 비용 고려 안 함.                                       | 편의성 중심 사용            |
